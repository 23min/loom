# loom self-host overlay

loom turned on loom (M-0017). This directory is the **entire** self-host footprint: three
properties whose Dafny lowerings model loom's own trust-critical plumbing, run through the
M-0016 runner via the opt-in `cargo loom` alias. It is off the default cargo graph (not a
workspace member) and removable without trace — `cargo build` / `cargo test` are byte-identical
whether or not it is present.

Each subdirectory is one **property** — a self-contained umbrella plus its formal lowering
(`model.dfy`) and, once verified, a generated `report.json` (git-ignored, regenerated by
`cargo loom`). The umbrellas declare their subject by pinned **symbol + version** (`subject-*`
fields), so each report records the exact Rust symbol its model stands for.

Properties:

- `dispatch-totality/` — every substrate routes to exactly one backend; none is silently
  unverified. Models `loom::backend::dispatch`.
- `parser-totality/` — every umbrella input yields a parsed umbrella or a typed rejection, never
  a panic. Models `loom::umbrella::parse`.
- `atomic-crash-safety/` — a crash mid-write leaves the report fully-old/absent or fully-new,
  never torn. Models `loom::atomic`.

**Honesty note.** A model mirrors its Rust subject *by reference* and is pinned to it by symbol;
it does not verify the Rust implementation directly (loom has no Rust→Dafny extraction —
ADR-0017). Two of loom's five load-bearing properties — gap-report writer↔reader equivalence
(D2) and reproducibility (G1) — are serde/meta facts whose honest home stays the Rust tests
M-0016 carries, not a Dafny model; they are deliberately not self-hosted here.
